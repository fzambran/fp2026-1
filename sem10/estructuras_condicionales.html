¿Qué son las Estructuras de Control?

01

Las estructuras de control determinan el orden de ejecución de las instrucciones en un programa.

En Python existen tres tipos principales:

  • Secuenciales: instrucciones ejecutadas línea a línea, de arriba hacia abajo
  • Condicionales: un bloque se ejecuta solo si se cumple una condición (if, elif, else)
  • Repetitivas: un bloque se repite mientras se cumpla una condición (for, while)

📌 En esta unidad nos enfocamos en las estructuras secuenciales y condicionales.

Estructura Secuencial

02

Es la forma más básica: las instrucciones se ejecutan una tras otra, en el orden en que aparecen en el código.

nombre = input("¿Cuál es tu nombre?: ")
edad   = int(input("¿Cuántos años tienes?: "))
print(f"Hola {nombre}, tienes {edad} años.")
print("Fin del programa.")
  • El programa siempre sigue el mismo camino
  • No hay bifurcaciones ni repeticiones
  • Todas las líneas se ejecutan exactamente una vez

Este flujo es el más simple, pero insuficiente cuando necesitamos tomar decisiones.

La Instrucción if

03

La instrucción if ejecuta un bloque de código solo si la condición es verdadera.

if condición: bloque de código (se ejecuta si True)

¿Cómo funciona?

  1. Python evalúa la condición
  2. Si es True → ejecuta el bloque indentado
  3. Si es False → omite el bloque y continúa

Ejemplo 1 — Verificación de Nombre

🐍

Contexto: Un sistema pide el nombre del usuario y verifica si tiene acceso.

# ejemplo1.py
mi_nombre = input("¿Cuál es tu nombre?: ")

if mi_nombre == "Felipe":
    print("¡Bienvenido!")
    print("Tienes acceso al sistema.")
else:
    print("¿Quién eres tú?")
    print("Acceso denegado.")

Análisis: Si el nombre ingresado es "Felipe" → saluda y da acceso. Cualquier otro nombre → deniega el acceso.

La Instrucción if / else

04

El bloque else define qué ocurre cuando la condición del if es falsa.

if condición: bloque A (True) else: bloque B (False)

  • Siempre se ejecuta exactamente uno de los dos bloques
  • else no lleva condición propia
  • Garantiza una respuesta para cualquier caso

Ejemplo 2 — Gatos o Perros

🐍

Contexto: El programa pregunta la preferencia del usuario y responde en consecuencia.

# ejemplo2.py
preferencia = input("¿Eres fanático de los gatos o de los perros?: ")

if preferencia == "gatos":
    print("¡Miau!")
else:
    print("¡Guau!")

Posibles resultados: → Si el usuario escribe "gatos" → imprime ¡Miau! → Cualquier otra respuesta → imprime ¡Guau!

La Indentación en Python

05

La indentación en Python no es opcional: es parte de la sintaxis y define qué instrucciones pertenecen a un bloque.

❌ Sin indentación — Error

if edad >= 18:
print("Eres mayor de edad")
# IndentationError

✅ Con indentación correcta

if edad >= 18:
    print("Eres mayor de edad")
    # 4 espacios obligatorios

Reglas de oro:

  • Usa 4 espacios por nivel de indentación (estándar PEP 8)
  • Sé consistente: no mezcles tabs y espacios en el mismo archivo
  • Todo el bloque debe tener la misma sangría

Errores Comunes de Indentación

⚠️

❌ Sangría inconsistente

if puntaje > 50:
    print("Aprobado")
  print("¡Felicidades!")
# IndentationError

❌ Falta el : al final

if puntaje > 50
    print("Aprobado")
# SyntaxError

✅ Correcto

if puntaje > 50:
    print("Aprobado")
    print("¡Felicidades!")

✅ Con else correcto

if puntaje > 50:
    print("Aprobado")
else:
    print("Reprobado")

La Instrucción if / elif / else

06

elif (else if) permite evaluar múltiples condiciones en cadena.

if condición1: bloque A elif condición2: bloque B elif condición3: bloque C else: bloque D

Reglas clave:

  • Python evalúa de arriba hacia abajo
  • Se ejecuta solo el primer bloque cuya condición sea True
  • El else captura todos los casos no cubiertos
  • Se pueden encadenar tantos elif como se necesite
  • El else es opcional, pero recomendable

Ejemplo 3 — Clasificación por Edad

🐍

# ejemplo3.py
edad = int(input("Ingresa tu edad: "))

if edad < 13:
    print("Eres un niño o niña")
elif edad < 20:
    print("Eres un adolescente")
elif edad < 65:
    print("Eres un adulto")
else:                             
    print("Eres un adulto mayor")

Corrección: El elif edad >= 65: original fue reemplazado por else:. Esto es más robusto: captura cualquier valor ≥ 65 y maneja valores inesperados (p. ej., edades negativas).

Ejemplo 4 — Días de la Semana

🐍

# ejemplo4.py 
dia = int(input("Ingresa un número del 1 al 7 para el día de la semana: "))

if dia == 1:
    print("Lunes")
elif dia == 2:
    print("Martes")
elif dia == 3:
    print("Miércoles")
elif dia == 4:
    print("Jueves")
elif dia == 5:
    print("Viernes")
elif dia == 6:
    print("Sábado")
elif dia == 7:
    print("Domingo")
else:
    print("Número de día inválido")

Corrección: Se agregó else al final. Sin él, ingresar un número fuera del rango 1–7 no produce ningún mensaje. Esto es una falla de validación de entrada.

Condicionales Anidados

07

Un condicional anidado es un if dentro de otro if. Permite tomar decisiones en múltiples niveles.

if condición_externa:
    if condición_interna:
        bloque A        # ambas condiciones son True
    else:
        bloque B        # solo la externa es True
else:
    bloque C            # la condición externa es False

Regla práctica: Cada nivel de anidamiento agrega 4 espacios de indentación. Se recomienda no superar 3 niveles de anidamiento para mantener la legibilidad del código.

Ejemplo 5 — Validación Académica

🐍

# ejemplo5.py
nota_presentacion = int(input("Ingresa tu nota de presentación (1-7): "))
asistencia        = int(input("Ingresa tu asistencia (0-100): "))

if nota_presentacion >= 4:
    if asistencia >= 70:
        print("Aprobado")
    else:
        print("Habla con tu profesor para revisar tu situación")
else:
    print("Reprobado")

Lógica: Solo se evalúa la asistencia si la nota de presentación es suficiente (≥ 4). Si la nota no alcanza el mínimo, el resultado es directo: reprobado.

Ejemplo 6 — Clasificación por Generación

🐍

# ejemplo6.py
anio = int(input("Ingresa tu año de nacimiento: "))

if 1965 <= anio <= 1980:
    gen = "Generación X"
elif 1981 <= anio <= 1996:
    gen = "Millennials"
elif 1997 <= anio <= 2012:
    gen = "Generación Z"
elif anio >= 2013:
    gen = "Generación Alfa"
else:
    gen = "Generación no identificada"

print(f"Perteneces a la {gen}")

Nota: Python permite condiciones del tipo 1965 <= anio <= 1980 (rango inclusivo), equivalente a anio >= 1965 and anio <= 1980.

Expresiones Condicionales

08

Python permite escribir condiciones simples en una sola línea usando la expresión condicional (también llamada operador ternario):

variable = valor_si_true if condición else valor_si_false

Forma tradicional:

if edad >= 18:
    estado = "mayor de edad"
else:
    estado = "menor de edad"

Expresión condicional:

estado = "mayor de edad" \
         if edad >= 18    \
         else "menor de edad"

Útil para asignaciones simples. Evitar en lógica compleja, ya que reduce la legibilidad.

Ejemplo 7 — Operador Ternario en Práctica

🐍

# Verificar si un número es par o impar
numero  = int(input("Ingresa un número entero: "))
paridad = "par" if numero % 2 == 0 else "impar"
print(f"El número {numero} es {paridad}.")
# Verificar mayoría de edad
edad   = int(input("Ingresa tu edad: "))
estado = "mayor de edad" if edad >= 18 else "menor de edad"
print(f"Eres {estado}.")
# Comparar dos números
a = int(input("Ingresa un número entero: "))
b = int(input("Ingresa otro número entero: "))

if b > a:
    print(f"{b} es mayor que {a}: {b} > {a}")
elif a > b:
    print(f"{a} es mayor que {b}: {a} > {b}")
else:
    print(f"{a} es igual a {b}: {a} == {b}")

Buenas Prácticas

09

✅ Hacer:

  • Usar 4 espacios para la indentación (nunca tabs)
  • Incluir else para manejar casos no previstos
  • Escribir condiciones claras y legibles
  • Limitar el anidamiento a máximo 3 niveles
  • Usar elif en lugar de if anidados cuando sea posible
  • Validar las entradas del usuario antes de procesarlas

❌ Evitar:

  • Mezclar tabs y espacios en el mismo archivo
  • Condiciones innecesariamente complejas en una línea
  • Olvidar los dos puntos : al final del if/elif/else
  • Usar operador ternario para lógica de múltiples pasos
  • Dejar bloques vacíos (usar pass si es necesario)
  • No manejar entradas fuera del rango esperado

Resumen de la Unidad

10

if / else Elige entre dos caminos posibles
if / elif / else Elige entre múltiples caminos posibles
Anidados Condiciones dentro de condiciones
Ternario x if c else y Asignación condicional en una línea
